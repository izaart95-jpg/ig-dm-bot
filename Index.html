Setup

Get up and running

Follow these steps to install, configure, and launch the bot.

1

Clone the repository

# Clone the repo git clone https://github.com/YOUR_USERNAME/YOUR_REPO.git cd YOUR_REPO
2

Install dependencies

The bot relies on instagrapi — an unofficial Instagram API wrapper for Python. Install it with pip:

# Install instagrapi (required) pip install instagrapi # Install remaining dependencies pip install requests

instagrapi handles login, session management, sending/receiving DMs, and proxy configuration. requests is used for LLM API calls.

3

Log in & save your session

Open login.py and replace the placeholder credentials with your Instagram account details, then run it. This dumps an authenticated session to session.json so the bot doesn't need your password every time.

# Edit login.py with your credentials username = "your_email@example.com" password = "YourPassword123" # Then run the login script python login.py

On success you'll see:

🔐 Logging in... ✅ Login successful! 💾 Session dumped to session.json

Important: Never commit session.json to a public repository. Add it to your .gitignore immediately.

4

Configure the bot

Open bot.py and update the config section at the top of the file. At minimum, set TARGET_USER_ID to the Instagram user ID you want the bot to chat with.

TARGET_USER_ID = 99123456789 # Replace with actual Instagram user ID POLL_INTERVAL = 3 # Seconds between message checks MAX_MSG_LENGTH = 900 # Max chars per DM (Instagram limit)
5

Run the bot

python bot.py

You should see output like:

🔍 Looking for DM thread with user... ✅ Thread found: 340282366841710300000000000000000000 ℹ️ 3 existing messages marked as read. 🎧 Listening for new messages... ============================================================
Optional

Proxy Setup

Avoid Instagram flagging your bot by routing traffic through a proxy.

Why use a proxy?

Instagram monitors for automated activity. If your bot logs in and sends messages from a server IP or an unfamiliar location, Instagram may temporarily lock the account or request verification. A residential or matching-location proxy makes the traffic appear as normal browser usage.

How to configure

Both login.py and bot.py have a commented-out proxy line. Uncomment it and provide your proxy URL:

# ─── In login.py ─── cl = Client() cl.set_proxy("http://ip:port") # HTTP proxy # cl.set_proxy("socks5://ip:port") # SOCKS5 proxy # cl.set_proxy("http://user:pass@ip:port") # Authenticated cl.login(username, password)
# ─── In bot.py ─── PROXY = "http://ip:port" cl = Client() cl.set_proxy(PROXY) # ... rest of setup

Tips

  • • Use a residential proxy — datacenter IPs are heavily flagged by Instagram. Residential or mobile proxies are far less likely to trigger security checks.
  • • Match your location — choose a proxy IP in the same country/region as your Instagram account to avoid "suspicious login location" alerts.
  • • Set the proxy in both files — login.py and bot.py create separate Client() instances, so you need to configure it in both places.
  • • Increase the poll interval — setting POLL_INTERVAL higher (e.g. 5–10 seconds) reduces API call frequency and looks more human-like.
Usage

How it works

login.py

Authenticates with Instagram and saves the session to session.json. Run this once (or whenever the session expires).

python login.py

bot.py

The main bot process. Loads the saved session, finds the target DM thread, and polls for new messages to respond to.

python bot.py

Message Flow

1 Bot polls the DM thread every POLL_INTERVAL seconds
2 New text message from target user is detected
3 A "🤔 AI is thinking…" placeholder DM is sent instantly
4 Full conversation history is sent to the LLM endpoint
5 LLM response is sent as a DM (auto-chunked if over 900 chars)
6 Response is appended to conversation memory for future context

Configuration

All configurable options in bot.py

Variable Default Description
TARGET_USER_ID — Instagram user ID of the person the bot should chat with
PROXY commented out HTTP/SOCKS5 proxy URL (e.g. "http://ip:port")
LLM_API_URL kilo.ai/openrouter Endpoint for the OpenAI-compatible chat completions API
LLM_MODEL stepfun/step-3.7-flash:free Model identifier passed to the LLM API
SYSTEM_PROMPT — System prompt that defines the AI's personality and behavior
POLL_INTERVAL 3 Seconds between each poll for new messages
MAX_MSG_LENGTH 900 Maximum characters per DM; long responses are auto-split
